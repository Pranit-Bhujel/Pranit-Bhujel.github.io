Multi-Line Statements and Strings¶

Understanding Multi-Line statements definitely makes us a better developer. Multi-Line statements/strings are generally used for improving the readability of the code. The interpreter doesn't bother if the given program is cramped in one line or extended to multiple lines, as long as the syntax is right.

First, we need to understand the difference between physical newline character and logical newline token.

. . .¶

Physical Newline Character: '\n' The character which is passed onto the program while we press enter to write a statement in the newline.

Logical Newline Token: Once we execute the program, the python interpreter breaks the program into logical lines of code and they are tokenized. These tokens/logical lines end with a logical newline token, which is then used by the python interpreter to execute the code.

Sometimes, physical newlines are ignored in order to combine multiple physical lines terminated by a logical newline token.

Now that we know what physical and logical newlines are, we can move forward to ways, the compiler converts physical newlines to logical newlines.

. . .¶

Conversion can be implicit or explicit. Implicit is when the compiler automatically converts to logical and Explicit is when we explicitly put '\' character to tell the compiler

Implicit Conversion¶

This kind of situation is automatically handled by the python interpreter. The following examples depict scenarios where we can use them.

In [1]:
# Writing list/tuple/dictionary/set/function arguments over multiple lines.

my_list = [1,
           2,
           3]

my_tuple = (1,
            2,
            3)

my_dictionary = {
    "a": 1,
    "b": 2,
    "c": 3
}

my_set = {1,
          2,
          3}

my_func(arg1,
        arg2,
        arg3...)

my_dictionary
Out[1]:
{'a': 1, 'b': 2, 'c': 3}

We can also pass arguments while writing across multiple lines. These are calles inline comments which are again only possible where implicit conversion is possible.

In [3]:
my_list = [1, #item1
           2, #item2
           3]

my_list
Out[3]:
[1, 2, 3]

Explicit Conversion¶

This is where the multi-line statements are not implicitly converted to a single logical line, and that is why we use explicit character '\' to make the statements as a multi-line statement.

In [4]:
# Example

x, y = 10, 20
a = x >= y
b = x <= y
c = x == y

if a \
   or b \
   or c:
    print('Explicit it is!')

Explicit it is!

Explicit Conversion doesn't support inline comments.

. . .¶

All this is fine, but why would we write simple lists over multiple lines?

  • To increase the readability of the code.
  • In cases where we have nested lists, dictionaries it makes much more sense to have a readable structure than just a continuous single line.
  • Also in scenarios when we have to write multiple conditions in a single if statement.
  • And many more...

Multi-line Strings¶

Multi-line strings can be written using three ' or " characters, both at the beginning and at the end. These strings retain the whitespaces and every other formatting used.

Basically, anything that you write and the way you write is printed as it is. So, they are mainly used for something called as docstrings.

In [6]:
a_string = '''This is just an 
example of 
a multi-line
string!'''

print(a_string)

This is just an 
example of 
a multi-line
string!

Multi-line strings and comments are different things, in a way that comments are stripped off and ignored during the program execution, but multi-line strings are parsed!
