    EXTRA SPACES¶

    When extra spaces creep in, they make the categorical variables dirty and irrelevant. The number of dummy variables might inconsistently increase, without any valuable addition to the output. Hence treating extra space remains a prominent task.

    Solution: The main idea remains to strip out the extra spaces. Let's see where extra spaces can occur.

    Example:

    In [16]:
    '''Using lstrip() or rstrip() strips the left or right spaces.'''

spaces = pd.Series(['Hello World!        ', '          Hello World!', '     Hello World!        ']).str.strip()

spaces

    UNNECESSARY FORMATTING¶

    Formatting can come in various forms, which will wipe off the good factor from a dataset. Given below are some of the formatting that can creep in with data.

    • Fonts (inc size, bold, underline, italic, strikethrough etc)
    • Borders (position, location, thickness, weight, color etc)
    • Fills (color, pattern)
    • Number formatting (inc custom number formats)
    • Alignment (top, left, centered across selection, indented, merged cells)
    • Cell Protection

    Solution: These formatting only prevail in file formats like Excel workbook (.xlsx), which can be converted to Comma Separated Values (.csv) for removing all the formatting. CSV files donot support formatting or formulas in the cells.

    If there are formatting like merged cells which is preventing the file from exporting to CSV, then one may use Excel to remove the error.

    STRUCTURAL ERRORS¶

    These are such errors which increases the inconsistent distribution of data. Structural errors are those that arise during measurement, data transfer, or other types of "poor housekeeping."

    This is mostly a concern for categorical features. Structural errors can show up in one or many forms given below:

    • Spelling mistakes
    • Unnecessary capitalization
    • Abbrevations
    • Other typo errors

    For an example, say you have created a Google Form for a survey of students. In that survey, you leave the University Name as a short answer option. Now if the University name is Harvard Business School, there can be following responses:

    • HBS
    • Harvard Business School
    • HARVARD BUSINESS SCHOOL
    • Harvard business school

    This will result in four different distribution for the same class. Let's do a practical example now.

    In [18]:
    '''In this example, I'll take number of tourists attraction for various places.'''

data = [['Darjeeling', 5], ['Darj', 2], ['darjeeling', 6], ['darj', 1], ['Siliguri', 4], ['siliguri', 5], ['Jalpaiguri', 9], ['Alipurduar', 15], ['Coochbehar', 11]]
df = pd.DataFrame(data, columns=['Places', 'Destinations'])

df
plt.bar(df['Places'], df['Destinations'])
plt.ylabel('Number of attractions')
plt.xticks(rotation =  45)
    Out[18]:
    ([0, 1, 2, 3, 4, 5, 6, 7, 8], <a list of 9 Text xticklabel objects>)

    INFERENCE

    We can observe that Darj, Darjeeling, darjeeling, darj are basically the same place, and so is siliguri & Siliguri. These errors not only increases the class distribution, but also inconsistent results. Now we'll look at various ways to remove these kind of structural errors.

    One way to tackle the problem is to replace the value cell wise, which can be achieved using the functions

    pd.loc[:, ['columns']] or pd.loc[:, ['columns']].

    Another way to handle the problem is to use the pd.replace() function. But remember not to overwrite the original DataFrame.

    In [19]:
    '''pd.replace() can be used both in the pandas DataFrame and pandas Series.'''
df['Places'] = df['Places'].replace(['darj', 'DARJEELING', 'darjeeling', 'Darj'], 'Darjeeling')
df['Places'] = df['Places'].replace(['siliguri'], 'Siliguri')

'Just for plotting'
plt.bar(df['Places'].unique(), df.groupby(['Places']).sum()['Destinations'], width = 0.5, color = 'orange')
plt.title('Clean Distribution')
    Out[19]:
    Text(0.5, 1.0, 'Clean Distribution')

    We can now see a much more clean distribution. Awesome :)

    UNWANTED OUTLIERS¶

    Outliers are those observations which seem unreal and does not follow the trend. Outliers can be of two types:

    • Univariate: Dealing with a single feature space.
    • Multivariate: Dealing with multiple dimensional features.

    Outliers can come in different flavours too, depending on the environment.

    • Point Outliers: Single data points which stands far from the rest of the distribution.
    • Contextual Outliers: Deals with various context, like punctuation symbols when doing text analysis, and background noise when doing speech recognition.
    • Collective Outliers: A collection of outliers in the data such that it may indicate the discovery of new phenomena.

    Common Causes of Outliers¶

    • Data entry errors
    • Measurement errors (instrument errors)
    • Experimental errors (data extraction / execution errors)
    • Intentional (dummy outliers to test detection methods)
    • Data processing errors (data manipulation fault)
    • Sampling errors (extracting or mixing data from various sources)
    • Natural (not an error, novelties in data)

    Novelties in a data is not an error, but a collection of data points which yield novel results.

    In the process of collection and manipulation of data, outliers can come and hide in multiple features, so it becomes very important to detect them.

    Always remember that the better the input data, the better the prediction.

    Outlier Detection Techniques¶

    There are many ways to detect outliers in data. Before going into the quest of finding outliers, it's important to have an answer to the following two question.

    Which and how many features am I taking into account to detect outliers ? (univariate / multivariate)

    Can I assume a distribution(s) of values for my selected features? (parametric / non-parametric)

    • Box Plot
    • Z-Score or Extreme Value Analysis (parametric)
    • Probabilistic and Statistical Modeling (parametric)
    • Linear Regression Models (PCA, LMS)
    • Proximity Based Models (non-parametric)
    • Information Theory Models
    • High Dimensional Outlier Detection Methods (high dimensional sparse data)

    Unlike all other methods above which detect outliers, there's a method to reduce the error caused by the outlier. It is called Minkowski error. We'll discuss that too.
